How To Install & Maintain Free TLS certificates Using acmetool with Nginx

Introduction

Let’s Encrypt is a free, automated, and open Certificate Authority that makes it possible to set up an HTTPS server and have it automatically obtain a browser-trusted certificate, without any human intervention. This is accomplished by running an Acme Protocol-compliant certificate management agent on the web server. While there are many such tools available, including Let's Encrypt's own "certbot", not all are created equal. One program in particular, acmetool, is especially simple to configure and feature-rich.

This tutorial provides instructions for the installation and maintenance of TLS certificates using "acmetool". Upon completion, users should have a valid, automatically-renewing certificate that is trusted by most modern web browsers.

Due to acmetool's built-in web server and ability to adapt to available challenge methods, an existing web server isn't necessary for development purposes. However, given that a TLS certificate serves no practical purpose without a method by which to serve it, reader's of this article will hence configure Nginx to reverse-proxy challenge requests to a non-privileged acmetool process listening on an internal port.

Note: While acmetool does maintain repositories for Ubuntu-, Debian-, and RPM-based distributions, the intent of this tutorial is to maintain cross-platform applicability. Users of any Ubuntu, Debian, Fedora, FreeBSD, or CentOS droplet can follow this tutorial with minimal substitutions, the most likely of which being Nginx configuration file locations.

Prerequisites

Although this article applies to many platforms and architectures, it assumes the following prerequisites for the sake of simplicity and readability.

  • A fresh Ubuntu 16.04 (64-bit) instance. You can learn how to set this up by following this initial server setup guide

  • A domain name with nameservers pointed to DigitalOcean nameservers (refer to this tutorial).

  • A web server listening on port 80 (optional). This tutorial assumes a basic Nginx setup, such as the one explained here.

Note: Ensure that the "sudo" package is installed (If using a distro other than the above requisite). It is required internally by acmetool.

Step 1 — Login as Root

ssh root@example.com

Step 2 - Download and install acmetool

wget "https://github.com/hlandau/acme/releases/download/v${acmever:=0.0.58}/acmetool-v${acmever}-${ostype:=linux}_${arch:=amd64}.tar.gz"
tar xf "acmetool-v${acmever}-${ostype}_${arch}.tar.gz"
cp "acmetool-v${acmever}-${ostype}_${arch}/bin/acmetool" /usr/local/bin

Step 3 — Add and Configure "acme" User

useradd -mU acme
mkdir -p ~acme/lib/acme ~acme/hooks
chown acme:acme -R ~acme
chmod 0750 ~acme/lib/acme ~acme/hooks

Step 4 — Run acmetool quickstart

sudo -u acme acmetool --state ~acme/lib/acme --hooks ~acme/hooks quickstart

You will be asked a series of questions. For the sake of this tutorial, use the following values.

  1. Enter "1" for live certificates.
  2. Enter "2" for PROXY method.
  3. Enter "y" to accept the Terms of Use.
  4. (Optionally) Enter your email address. If you choose to provide an email, you will be notified about expiring certificates, as well as other useful information about Let's Encrypt.

Note: For in-depth explainations of the various options, please refer to the acmetool userguide.
